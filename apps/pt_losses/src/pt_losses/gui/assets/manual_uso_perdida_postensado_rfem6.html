Manual de uso

Aplicación: PérdidaPostensadoRFEM6

Propósito: calcular pérdidas de postensado y convertir los resultados en deformaciones axiales equivalentes para su aplicación en RFEM 6.

Deslinde de responsabilidades
Los valores obtenidos con esta herramienta deben ser revisados, verificados y validados por un ingeniero estructural competente antes de su uso en proyecto, documentación, cálculo ejecutivo o construcción. La aplicación no reemplaza el criterio profesional ni la verificación normativa correspondiente.

1. Introducción

PérdidaPostensadoRFEM6 es una herramienta para asistir en el cálculo de pérdidas de postensado de tendones y en la obtención de deformaciones axiales equivalentes para RFEM 6.

La aplicación permite dos modos de trabajo:

  • Modo manual, ingresando datos directamente en la pestaña Cálculo.
  • Modo integrado con RFEM 6, leyendo automáticamente el modelo, los tendones y los materiales desde la API de Dlubal.

2. Resultados que entrega la herramienta

  • Tensión máxima utilizable del acero.
  • Estado inicial de tesado.
  • Estado final con pérdidas.
  • Fuerza inicial y final por tendón y total.
  • Desglose de pérdidas inmediatas y diferidas.
  • Valores equivalentes para RFEM 6 en T = 0 y T = infinito.

3. Requisitos

  • Windows.
  • Python 3.11+ o el ejecutable PerdidaPostensadoRFEM6.exe.
  • RFEM 6 instalado para el flujo integrado.
  • Servicio gRPC/API habilitado en RFEM.
  • API key válida de Dlubal.
Seguridad de la API key
El ejecutable distribuido no debe incluir una API key embebida. La clave se configura por primera vez desde Opciones > Configurar API RFEM... y luego queda guardada localmente en la computadora de uso hasta que se cambie nuevamente.

5. Flujo lógico con RFEM 6

  1. Abra RFEM 6.
  2. En la pestaña RFEM 6 pulse LEER MODELO DE RFEM 6.
  3. La aplicación intenta primero leer el modelo activo de RFEM.
  4. Si RFEM no tiene un modelo abierto, la aplicación le pide al usuario seleccionar un archivo .rf6.
  5. Una vez seleccionado, la aplicación lo abre en RFEM desde el propio script y luego lee sus datos automáticamente.
  6. La interfaz completa los IDs de members tendón, cantidad de members, cantidad de cordones y propiedades principales del modelo cuando están disponibles.
  7. Revise o complete los campos de la pestaña Cálculo.
  8. Pulse Calcular.
  9. Revise tensiones, fuerzas, pérdidas y lectura para RFEM 6.
  10. Si el resultado es correcto, pulse Aplicar en RFEM.

6. Estructura de la interfaz

6.1 Pestaña RFEM 6

  • Botón principal LEER MODELO DE RFEM 6.
  • Línea Modelo cargado con el nombre del modelo activo o abierto.
  • IDs de members tendón, Cantidad de members y Cantidad de cordones.
  • Botones: Probar conexión, Volver a leer modelo, Autocompletar y Aplicar en RFEM.

6.2 Pestaña Cálculo

Permite ingresar o corregir manualmente:

  • Propiedades del acero de postensado: Ep, fpk, fp01k, Ap.
  • Propiedades del hormigón: Ec, fc.
  • Datos geométricos, de tesado y de pérdidas.

6.3 Panel de resultados

  • Tarjetas de datos técnicos.
  • Resumen ejecutivo.
  • Desglose de pérdidas en porcentaje para presentación.
  • Lectura para RFEM 6 con scroll vertical.
  • Botón Exportar CSV.

7. Integración con la API de Dlubal

La integración utiliza la API oficial de Dlubal para RFEM 6. El flujo implementado en la aplicación sigue esta lógica:

  1. Conectarse a RFEM mediante gRPC usando la API key configurada.
  2. Consultar si existe un modelo activo.
  3. Si existe, usar ese modelo.
  4. Si no existe, abrir el archivo .rf6 seleccionado por el usuario.
  5. Leer members tipo tendón, sección, material del tendón y material de hormigón principal.
  6. Crear los casos de carga necesarios y aplicar las deformaciones axiales equivalentes.

Documentación oficial consultada:

  • Documentación general de la API de Dlubal
  • Quick Start
  • Access Control / API Keys

8. Aplicación automática en RFEM

La aplicación automática trabaja actualmente sobre members/barra tipo tendón. El modelo global puede contener superficies, pero la deformación axial equivalente se aplica al tendón modelado como member.

La numeración de casos de carga y cargas de miembro se resuelve automáticamente buscando el próximo número libre del modelo.

9. Notas sobre fiabilidad del resultado

Esta herramienta es adecuada para estimación preliminar, comparación de escenarios y generación de estados equivalentes para RFEM. No sustituye una verificación normativa completa ni un modelo detallado de pérdidas secuenciales.

Si las pérdidas totales resultan inusualmente altas, revise especialmente:

  • anchorage_slip_mm respecto de tendon_length.
  • mu_fric, k_wobble y theta_total.
  • concrete_stress_at_tendon y creep_coeff.
  • relaxation_loss_ratio.

10. Solución de problemas

  • No se conecta a RFEM: revise que RFEM esté abierto, que el servicio API esté activo y que el puerto sea correcto.
  • No hay modelo activo: pulse LEER MODELO DE RFEM 6 y seleccione el archivo cuando se lo solicite.
  • No se detectan tendones: verifique que el tendón esté modelado como member/barra y no solo como superficie.
  • Faltan datos automáticos: revise si la sección o el material del tendón exponen esas propiedades en la API del modelo.
