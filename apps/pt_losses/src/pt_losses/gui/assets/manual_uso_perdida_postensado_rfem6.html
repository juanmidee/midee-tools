Manual de Uso

Aplicación: PérdidaPostensadoRFEM6

Propósito: calcular pérdidas de postensado y convertir los resultados en deformaciones axiales equivalentes para su aplicación en RFEM 6.

Deslinde de responsabilidades
Los valores obtenidos con esta herramienta deben ser revisados, verificados y validados por un ingeniero estructural competente antes de su uso en proyecto, documentación, cálculo ejecutivo o construcción. La aplicación no reemplaza el criterio profesional ni la verificación normativa correspondiente.

1. Introducción

PérdidaPostensadoRFEM6 es una herramienta para asistir en el cálculo de pérdidas de postensado de tendones y en la obtención de las deformaciones axiales equivalentes que luego pueden aplicarse en RFEM 6.

La aplicación permite dos modos de trabajo:

  • Modo manual, ingresando los datos del caso directamente en la pestaña Cálculo.
  • Modo integrado con RFEM 6, leyendo automáticamente el modelo, los tendones y los materiales desde la API de Dlubal.

2. Resultados que entrega la herramienta

  • Tensión máxima utilizable del acero.
  • Tensión inicial de tesado.
  • Tensión final a tiempo infinito.
  • Fuerza inicial y final por tendón y total.
  • Desglose de pérdidas inmediatas y diferidas.
  • Valores equivalentes para RFEM 6 en T = 0 y T = infinito.

3. Requisitos de uso

3.1 Requisitos generales

  • Sistema operativo Windows.
  • Python 3.11+ para ejecutar la versión sin compilar o el ejecutable PerdidaPostensadoRFEM6.exe.
  • Permisos para leer modelos, archivos JSON/YAML y exportar resultados.

3.2 Requisitos para integración con RFEM 6

  • RFEM 6 instalado y abierto.
  • Servicio gRPC/API habilitado en RFEM.
  • API key válida de Dlubal.
  • Modelo .rf6 accesible desde la PC.
Seguridad de la API key en la distribución
El ejecutable distribuido no debe incluir ninguna API key embebida. La aplicación queda vacía al distribuirse y la API key se configura por primera vez desde Opciones > Configurar API RFEM.... Una vez cargada, queda guardada localmente en la computadora de uso hasta que se la cambie nuevamente.

6. Flujo recomendado de uso

  1. Abra la aplicación.
  2. Si usará RFEM, abra RFEM 6 y cargue el modelo correspondiente.
  3. Vaya a Opciones > Configurar API RFEM... y pegue la API key si aún no está guardada.
  4. En la pestaña RFEM 6, seleccione el modelo .rf6.
  5. Pulse Leer modelo RFEM para detectar los tendones y leer propiedades del material.
  6. Revise o complete los campos necesarios en la pestaña Cálculo.
  7. Pulse Calcular.
  8. Revise las tensiones, fuerzas, pérdidas y la lectura de RFEM 6.
  9. Si el resultado es correcto, pulse Aplicar en RFEM.
  10. Si desea un archivo externo, utilice Exportar CSV.

7. Uso de la pestaña Cálculo

En esta pestaña pueden ingresarse o corregirse los parámetros del caso:

  • Propiedades del acero de postensado: Ep, fpk, fp01k, Ap.
  • Propiedades del hormigón: Ec, fc.
  • Datos geométricos y de tesado.
  • Pérdidas por rozamiento, anclaje, acortamiento elástico, fluencia, retracción y relajación.

Botones principales:

  • Calcular: ejecuta el cálculo con los datos actuales.
  • Restablecer: vuelve a los valores base del programa.
  • Cargar archivo: permite abrir un archivo JSON o YAML de entrada.
  • Cargar ejemplo: carga el ejemplo provisto con la aplicación.

9. Qué se lee automáticamente desde RFEM

  • Members tipo tendón.
  • Cantidad de members tendón.
  • Cantidad de cordones.
  • Longitud media del tendón.
  • Material del tendón y propiedades relevantes del acero.
  • Material principal de hormigón del modelo, cuando se puede identificar de forma consistente.

Nota: actualmente la aplicación trabaja principalmente con tendones modelados como members. Las superficies candidatas se informan en la lectura del modelo, pero no se aplican automáticamente.

10. Integración API con RFEM 6

La integración con RFEM 6 de esta aplicación sigue el flujo general documentado por Dlubal para su API gRPC. A continuación se resumen los puntos más relevantes para el usuario final.

10.1 Autenticación con API key

Según la documentación oficial de autenticación de Dlubal, el acceso a la API se realiza mediante una API key. Dlubal indica que la clave puede pasarse directamente al inicializar el cliente o bien resolverse por nombre desde un archivo config.ini. La documentación también señala que el uso de un api_key_name es el método preferido cuando se comparten scripts, ya que ayuda a proteger el valor directo de la clave.

En esta aplicación, por motivos de comodidad para el usuario, la API key se configura desde el menú Opciones y se guarda localmente en la computadora de uso. Esto permite que, después de la primera configuración, no sea necesario volver a pegar la clave en cada ejecución.

10.2 Generación y resguardo de la API key

La documentación oficial indica que las API keys se administran desde Extranet > API & Cloud. También advierte que, una vez generada la clave, debe copiarse y almacenarse en forma segura de inmediato, ya que por motivos de seguridad no volverá a mostrarse completamente.

10.3 Puesta en marcha del servidor API

Dlubal documenta que el servidor gRPC se inicia por defecto junto con la interfaz gráfica de RFEM/RSTAB. También explica que el puerto puede revisarse y modificarse en Options > Program Options > API Service | gRPC. En la mayoría de los casos prácticos se utiliza el puerto 9000.

Esto significa que, para trabajar normalmente con esta aplicación, basta con abrir RFEM 6 y verificar que el servicio API esté disponible en el puerto configurado.

10.4 Inicio rápido y cliente Python

La guía oficial de inicio rápido de Dlubal indica el uso de Python 3.10 o superior y la instalación del cliente oficial mediante pip install dlubal.api. También muestra un ejemplo básico de conexión usando:

from dlubal.api import rfem
with rfem.Application(api_key_name='key') as rfem_app:

Esta aplicación encapsula ese flujo y evita que el usuario tenga que programar manualmente la conexión.

13. Mensajes y comportamientos importantes

  • La API key sí se guarda localmente hasta que se cambie.
  • El ejecutable distribuido no incorpora la API key del equipo donde fue compilado.
  • El modelo RFEM seleccionado no se recuerda al reiniciar la aplicación.
  • Si RFEM está cerrado, la app mostrará un error de conexión gRPC.
  • Si no hay resultados calculados, no se permitirá exportar ni aplicar en RFEM.

14. Solución de problemas

Problema Acción recomendada
No conecta con RFEM Verifique que RFEM esté abierto, que el servicio gRPC esté activo y que el puerto sea el correcto.
No encuentra tendones Revise el tipo de member, el nombre y la definición del tendón dentro del modelo. La aplicación contempla variantes en español e inglés.
Los valores del material no coinciden Use Leer modelo RFEM y verifique que el tendón tenga su sección y material correctamente definidos en RFEM.
Las deformaciones en RFEM parecen desproporcionadas Revise la unidad RFEM y el factor de escala. La configuración recomendada actual de la interfaz es adimensional.

15. Fórmulas implementadas

sigma_max = min(0.80 · fpk, 0.94 · fp01k)
sigma_0 = mu_tesado · sigma_max
eta_fr = 1 - exp(-(mu_fric · theta_total + k_wobble · tendon_length))
eta_total = min(eta_fr + eta_anc + eta_el + eta_rel + eta_flu + eta_ret, 0.99)
sigma_inf = sigma_0 · (1 - eta_total)
T0_percent = -(sigma_0 / Ep) · 100
Tinf_percent = -(sigma_inf / Ep) · 100
